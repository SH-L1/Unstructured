# 최종 데이터 범위와 사용 방식

이 문서는 현재 단계에서 실제 개발에 사용할 데이터 범위를 확정한다. 부족한 데이터는 일단 제외하고, 아래 API 키와 다운로드 완료 데이터만 기준으로 개발한다.

DB 테이블 구성과 실제 파일 배치 경로는 [current-db-and-file-layout.md](current-db-and-file-layout.md)를 따른다.

## 기본 원칙

- API 키 자체는 DB에 저장하지 않는다.
- API 응답과 다운로드 파일은 수집 후 DB에 적재한다.
- DB에는 원천명, 수집 시각, 원문 해시, 버전, 관할, 검증 상태, 법적 근거 허용 여부를 함께 저장한다.
- 법적 근거로 사용할 수 있는 데이터는 국가법령정보 API에서 가져온 국가법령 중 검증을 통과한 조항으로 제한한다.
- 아산시 자치법규는 국가법령정보 API로 수집하되, 현재 단계에서는 참고/부서 확인용으로만 사용한다.
- 공공데이터, SGIS, 새올, AIHub, 민원편람, 조직도는 절차·위치·배정·문체·평가 보조 자료이며 법적 근거가 아니다.
- LLM은 좌표나 GeoJSON을 생성하지 않는다. 위치와 좌표는 SGIS, 주소정보, 공공데이터포털 공간 데이터에서만 온다.

## API 키 사용 데이터

### SGIS

키 변수:

- `SGIS_CONSUMER_KEY`
- `SGIS_CONSUMER_SECRET`
- `SGIS_ACCESS_TOKEN`: normally empty; the ingestion code issues and refreshes it from the consumer credentials.

가져올 데이터:

- 아산시 행정구역 경계
- 읍면동/법정동 경계
- 행정구역 코드
- 경계 geometry 또는 GeoJSON
- 중심점, bbox 계산용 좌표

사용 방식:

- 위치 후보가 아산시 관할인지 판정한다.
- 민원 위치가 어느 읍면동/법정동에 속하는지 확인한다.
- `NEEDS_JURISDICTION`, `NEEDS_LOCATION` 차단 여부를 판단한다.
- 대시보드에서 위치 후보와 관할 확인에 사용한다.

저장 위치:

- `spatial_source_registry`
- `spatial_admin_boundaries`

법적 근거 사용:

- 불가

### 공공데이터포털 공간/시설 데이터

키 변수:

- 기본 재사용: `COMPLAINT_BIGDATA_API_KEY`, `COMPLAINT_BIGDATA_SERVICE_KEY`, `POLICY_QNA_API_KEY`
- 선택 override: `SPATIAL_DATA_GO_KR_SERVICE_KEY`

대상 API:

- 전국도시공원정보표준데이터
- 전국주차장정보표준데이터
- 행정안전부_CCTV정보 조회서비스
- 전국주정차금지(지정)구역표준데이터

가져올 데이터:

- 도시공원명, 공원구분, 소재지 주소, 관리기관, 위도, 경도
- 주차장명, 주차장 유형, 운영 시간, 요금 정보, 소재지 주소, 관리기관, 위도, 경도
- CCTV 설치 목적, 설치 위치, 카메라 대수, 관리기관, 위도, 경도
- 주정차 금지구역 도로명, 구간, 지정 사유, 운영 시간, 관리기관, 위도, 경도 또는 구간 geometry

사용 방식:

- 불법주정차 민원의 단속 가능 구간과 주변 주차장 정보를 확인한다.
- 공원/녹지 관련 민원의 시설 위치와 관리기관을 확인한다.
- 안전 민원의 CCTV 존재 여부와 주변 안전 시설을 참고한다.
- 위치 후보 점수를 계산하고 사람 확인 대상 후보로 제시한다.

저장 위치:

- `spatial_source_registry`
- `spatial_facilities`
- `spatial_parking_restrictions`
- `spatial_location_resolution_runs`
- `spatial_location_candidates`

법적 근거 사용:

- 불가

### 국민권익위원회 민원정책 질의응답조회서비스

키 변수:

- `POLICY_QNA_API_KEY`

가져올 데이터:

- 민원 정책 Q&A 제목
- 질의 내용
- 답변 내용
- 관련 분류
- 원천 URL 또는 원천 식별자
- 수집 시각과 원문 해시

사용 방식:

- 민원 처리 절차 설명 참고 자료로 사용한다.
- 답변 초안의 절차 안내, 안내 문구, 상담 패턴 참고에 사용한다.
- 법적 판단이나 최종 근거로 쓰지 않는다.

저장 위치:

- `source_registry`
- `knowledge_documents`
- `knowledge_purpose`
- `data_mart_ingestion_runs`
- `data_mart_raw_records`
- `data_mart_normalized_records`

목적:

- `PROCEDURE`

법적 근거 사용:

- 불가

### 국민권익위원회 민원빅데이터 분석정보 API 2022~2025

키 변수:

- `COMPLAINT_BIGDATA_API_KEY`
- `COMPLAINT_BIGDATA_SERVICE_KEY`

가져올 데이터:

- 민원 키워드
- 키워드 순위
- 키워드 추이
- 민원 분류
- 기관별/지역별 통계
- 유사 사례성 분석 정보
- 기간별 빈도
- 민원 이슈 흐름

사용 방식:

- 민원 유형 분류 보조에 사용한다.
- 담당 부서 추천의 참고 신호로 사용한다.
- 최근 민원 경향과 표현 패턴을 참고한다.
- 법적 근거, 사실 인정, 최종 판단에는 사용하지 않는다.

저장 위치:

- `source_registry`
- `knowledge_documents`
- `knowledge_purpose`
- `historical_complaints` 또는 분석용 mart 테이블
- `data_mart_ingestion_runs`
- `data_mart_raw_records`
- `data_mart_normalized_records`

목적:

- `HISTORICAL_CASE`

법적 근거 사용:

- 불가

### 국가법령정보

키 변수:

- `LAW_API_OC`

가져올 데이터:

- 국가법령 목록
- 국가법령 본문
- 조문 단위 본문
- 시행일
- 공포일
- 개정/폐지 상태
- 법령 ID
- 조항 번호
- 법령 원문 URL
- 아산시 자치법규 본문과 조항

사용 방식:

- 국가법령은 검증을 통과한 조항만 법적 근거로 사용한다.
- 아산시 자치법규는 수집하되 현재 단계에서는 참고/부서 확인용으로만 사용한다.
- 초안의 모든 법적 주장은 `evidence_snapshots`에 복사된 공식 조항과 연결되어야 한다.

저장 위치:

- `source_registry`
- `legal_document_versions`
- `legal_provisions`
- `knowledge_documents`
- `knowledge_purpose`
- `evidence_snapshots`

목적:

- 국가법령: `OFFICIAL_LAW`
- 아산시 자치법규: `LOCAL_ORDINANCE_REFERENCE`

법적 근거 사용:

- 국가법령: 가능, 단 검증 통과 필수
- 아산시 자치법규: 현재 단계 불가

## 제외 데이터

아래 데이터는 현재 단계에서 제외한다.

- 아산시 전체 고시공고
- 내부 GIS 도로관리대장, 상하수도, 하수관거, 맨홀, 배수로 자료
- 축산시설 별도 인허가 대장
- 공동주택 별도 관리대장
- 도로안전표지, VMS, 가로등/보안등 별도 데이터 중 아직 확보하지 않은 자료

제외 이유:

- 현재 확보한 데이터만으로 개발 범위를 고정하기 위함
- 개인정보와 개별 처분 문서 유입 위험을 줄이기 위함
- 검색 오염을 줄이고 첫 구현 범위를 안정화하기 위함

## 구현 우선순위

1. 국가법령정보 API 적재와 조항 검증
2. SGIS 경계와 건물DB 주소 포인트 적재
3. 공공데이터포털 공원/주차장/CCTV/주정차금지구역 적재
4. 아산시 조직도 적재와 부서 추천 규칙 생성
5. 민원편람 `PROCEDURE` 적재
6. 새올 공개 상담민원 비식별 이력 적재
7. AIHub 데이터는 운영 근거가 아니라 평가/튜닝 후보로 분리

## 초안 생성 시 사용 규칙

- 법적 주장: 국가법령정보 API에서 수집한 검증 조항만 사용한다.
- 절차 안내: 민원편람, 민원정책 Q&A, 민원사무서식 자료를 사용한다.
- 담당 부서 추천: 아산시청 조직도, 새올 비식별 이력, 민원빅데이터 경향을 사용한다.
- 위치 확인: SGIS, 건물DB, 공공데이터포털 공간 데이터를 사용한다.
- 문체 참고: 새올 공개 상담민원과 공공 민원 상담 LLM 데이터를 사용한다.
- 첨부/문서 이해: 문서 이해 기반 시각요소 생성 데이터를 학습/평가 후보로 사용한다.
- 최종 승인과 완료: 항상 사람이 수행한다.
